## üìã Documentation Standards

### File Naming
- Use UPPERCASE for document names
- Use underscores for multi-word names
- Use .md extension

### Document Structure
- Start with a clear title
- Include table of contents for long documents
- Use proper heading hierarchy
- Include code examples where applicable
- Add navigation links

### Maintenance
- Update [Changelog](./CHANGELOG.md) for all changes
- Keep [Implementation History](./IMPLEMENTATION_HISTORY.md) current
- Update [Translation Status](./TRANSLATION_STATUS.md) when adding languages
